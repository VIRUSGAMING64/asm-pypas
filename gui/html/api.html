api reference
api
Base URL
http://localhost:8000
POST /api/run
Body JSON: {"name": "file.asm", "code": "..."}
Ejecuta el código y devuelve el resultado del intérprete.
Errores comunes: 400, 404, 500.
POST /api/save
Body JSON: {"name": "file.asm", "code": "..."}
Guarda el archivo en memoria y en disco.
Respuesta: {"status":"ok"}.
GET /api/getcode?name=<file>
Query param: name
Devuelve el contenido de un archivo concreto.
Respuesta: {"status":"ok","code":"..."}.
POST /api/getcode?name=<file>
Query param: name
La misma ruta también acepta POST por compatibilidad.
Respuesta: {"status":"ok","code":"..."}.
GET /api/initcodes
Sin parámetros
Lista los nombres disponibles dentro de codes/.
Respuesta: {"status":"ok","names":[...]}
GET /api/newcode?name=<file>
Query param: name
Crea una entrada vacía y la deja lista para editar.
Respuesta: {"status":"ok","code":""}.
GET /api/delcurr?name=<file>
Query param: name
Elimina el archivo actual del almacenamiento.
Respuesta: {"status":"ok"} o {"status":"fail"}.
Respuestas
Éxito: {"status":"ok"} o {"status":"ok","code":"..."}.
Error: {"status":"fail","message":"..."}.
Validación
Los nombres de archivo se validan para evitar path traversal.
El servidor limita el payload JSON a 128 MB de código.
